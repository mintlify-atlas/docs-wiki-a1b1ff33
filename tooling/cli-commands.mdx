Every ABP CLI command is an `IConsoleCommand` implementation registered in `AbpCliOptions.Commands` by `AbpCliCoreModule`. Commands receive a `CommandLineArgs` object that carries the parsed command name, positional target, and an options dictionary. The sections below document each command's purpose, key flags, and what it does under the hood.

## Command quick-reference

| Command | Class | Short description |
|---|---|---|
| `new` | `NewCommand` | Scaffold a new solution from a startup template |

## `new`

**Class:** `NewCommand` — extends `ProjectCreationCommandBase`

Scaffolds a new solution from an ABP startup template. Internally it builds a `ProjectBuildArgs` object and calls `TemplateProjectBuilder.BuildAsync(args)`, which downloads the template ZIP from abp.io (or a local path), runs the `ProjectBuildPipeline`, and extracts the result to disk.

```
abp new <project-name> [options]
```

### Key options

| Flag | Default | Description |
|---|---|---|
| `-t\|--template` | `app` | Template name (`app`, `app-nolayers`, `module`, `console`, …) |
| `-u\|--ui` | `mvc` | UI framework (`mvc`, `angular`, `blazor`, `blazor-server`, `blazor-webapp`, `maui-blazor`, `none`) |
| `-m\|--mobile` | — | Mobile framework (`none`, `react-native`, `maui`) |
| `-d\|--database-provider` | `ef` | `ef` or `mongodb` |
| `--dbms` | SQL Server | `sqlserver`, `mysql`, `postgresql`, `oracle`, `sqlite` |
| `-o\|--output-folder` | current dir | Where to extract the solution |
| `-v\|--version` | latest | ABP version to use |
| `--preview` | false | Use latest pre-release version |
| `--tiered` | false | Separate auth-server and API host |
| `--no-ui` | false | Skip all UI projects (module template) |
| `--theme` | `leptonx-lite` | UI theme (`leptonx-lite`, `leptonx`, `basic`, `lepton`) |
| `-cs\|--connection-string` | — | Override the default connection string |
| `-csf\|--create-solution-folder` | true | Create a containing folder for the solution |
| `-ts\|--template-source` | — | Local or network path to use as template source |
| `--local-framework-ref --abp-path` | — | Keep local `<ProjectReference>` entries instead of NuGet references |
| `-sib\|--skip-installing-libs` | false | Skip `abp install-libs` after creation |
| `-sb\|--skip-bundling` | false | Skip `abp bundle` for Blazor WASM |
| `-sc\|--skip-cache` | false | Always download a fresh template archive |
| `--no-random-port` | false | Keep template default localhost ports |
| `--separate-auth-server` | false | Separate the auth server from the API host |

### Post-creation steps

After extraction, `NewCommand.ExecuteAsync` automatically:

1. Creates OpenIddict PFX certificate files (`CreateOpenIddictPfxFilesAsync`)
2. Runs `dotnet build` for microservice service templates (`RunGraphBuildForMicroserviceServiceTemplate`)
3. Creates initial EF Core migrations (`CreateInitialMigrationsAsync`)
4. Runs `abp install-libs` (unless `--skip-installing-libs`)
5. Runs `abp bundle` for Blazor WASM (unless `--skip-bundling`)
6. Opens the relevant documentation page in the browser (unless `--no-open-web-page`)

## `add-module`

**Class:** `AddModuleCommand`

Adds a multi-package ABP module to an existing solution. It must be run from a directory containing a `.sln` or `.slnx` file (or you can pass `-s`).

```
abp add-module <module-name> [options]
```

Internally, `SolutionModuleAdder.AddAsync` is called. It:

1. Fetches module metadata from abp.io
2. Identifies which NuGet packages the module exposes
3. Matches each package to the corresponding project in the solution (e.g., `.Application` package → `.Application.csproj`)
4. Adds each NuGet reference and injects `[DependsOn(...)]` attributes via `ModuleClassDependcyAdder`
5. Optionally creates a new EF Core migration

### Key options

| Flag | Description |
|---|---|
| `--new` | Creates a fresh local module instead of referencing published packages |
| `--with-source-code` | Downloads the module source instead of using NuGet references |
| `--add-to-solution-file` | Adds the downloaded source projects to the `.sln` (only with `--with-source-code`) |
| `--skip-db-migrations` | Skips EF Core migration creation (always skipped with `--new`) |
| `-s\|--solution` | Path to the solution file |
| `-v\|--version` | Specific ABP version; defaults to the solution's current ABP version |
| `-sp\|--startup-project` | Relative path to the startup project for migration commands |
| `--skip-opening-documentation` | Don't open the module's documentation URL after adding |

<Note>
Some modules are explicitly blocked in `AbpCliOptions.DisabledModulesToAddToSolution` and cannot be added with this command. Current entries include `Volo.Abp.LeptonXTheme.Pro` and `Volo.Abp.LeptonXTheme.Lite`.
</Note>

## `add-package`

**Class:** `AddPackageCommand`

Adds a single ABP NuGet or NPM package to a project. The command detects package type: names starting with `@` are treated as NPM packages; everything else is NuGet.

```
abp add-package <package-name> [options]
```

For NuGet packages, `ProjectNugetPackageAdder` adds the `<PackageReference>` to the `.csproj` and injects the corresponding `[DependsOn(typeof(...))]` attribute to the ABP module class. For NPM packages, `ProjectNpmPackageAdder` updates `package.json`.

### Key options

| Flag | Description |
|---|---|
| `-p\|--project` | Target `.csproj` file (NuGet only) |
| `-s\|--solution` | Solution file to resolve project from |
| `-v\|--version` | Package version; defaults to solution's ABP version |
| `--with-source-code` | Download source instead of referencing the NuGet package |
| `--add-to-solution-file` | Include downloaded source project in `.sln` |
| `-ad\|--angular-directory` | Angular project directory (NPM packages) |

## `generate-proxy`

**Class:** `GenerateProxyCommand` — extends `ProxyCommandBase<GenerateProxyCommand>`

Generates typed client-side service proxies (TypeScript/Angular, plain JavaScript, or C#) by reading the `/api/abp/api-definition` endpoint from a running ABP application.

```
abp generate-proxy -t <type> [options]
```

`ProxyCommandBase.ExecuteAsync` uppercases the `-t` value and looks up the generator type in `AbpCliServiceProxyOptions.Generators`, then calls `IServiceProxyGenerator.GenerateProxyAsync(args)`.

### Generator types

| `-t` value | Generator class | Output |
|---|---|---|
| `ng` | `AngularServiceProxyGenerator` | Angular services + TypeScript DTOs |
| `js` | `JavaScriptServiceProxyGenerator` | Plain `.js` proxy file |
| `csharp` | `CSharpServiceProxyGenerator` | C# client classes + DTO types |

### Key options

| Flag | Description |
|---|---|
| `-t\|--type` | `ng`, `js`, or `csharp` (**required**) |
| `-u\|--url` | API definition URL (e.g., `https://localhost:44302/`) |
| `-m\|--module` | Backend module name (default: `app`) |
| `-wd\|--working-directory` | Execution directory (default: current directory) |
| `--folder` | Folder for generated C# code (default: `ClientProxies`) |
| `-o\|--output` | Output file/folder for JS proxies |
| `--without-contracts` / `-c` | C# only — skip DTO and interface generation |
| `-a\|--api-name` | Angular API endpoint name (default: `default`) |
| `-s\|--source` | Angular source project name |
| `--target` | Angular target project name |
| `-p\|--prompt` | Angular only — interactively prompt for missing options |
| `-st\|--service-type` | Filter to `application`, `integration`, or all services |
| `-ep\|--entry-point` | Override entry point resolution |

## `remove-proxy`

**Class:** `RemoveProxyCommand` — extends `ProxyCommandBase<RemoveProxyCommand>`

Deletes previously generated proxy files. Accepts the same `-t`, `-m`, `-o`, and `--folder` flags as `generate-proxy`. The generator's `GenerateProxyAsync` is called in "remove" mode (the `CommandName` is `"remove-proxy"`).

```
abp remove-proxy -t ng
abp remove-proxy -t csharp --folder MyProxies/InnerFolder
```

## `bundle`

**Class:** `BundleCommand`

Scans a Blazor WebAssembly or MAUI Blazor project for ABP module style/script contributions and rewrites `index.html` with correct `<link>` and `<script>` tags. Delegates to `IBundlingService.BundleAsync`.

```
abp bundle [options]
```

| Flag | Default | Description |
|---|---|---|
| `-wd\|--working-directory` | current dir | Project root to scan |
| `-t\|--project-type` | `webassembly` | `webassembly` or `maui-blazor` |
| `-f\|--force` | false | Force project rebuild before bundling |

---

## `build`

**Class:** `BuildCommand`

Smart incremental build for ABP repository development. Reads a `abp-build-config.json` configuration file, determines which projects changed (via Git diff), resolves transitive dependencies, and invokes `dotnet build` only on affected projects.

```
abp build [options]
```

| Flag | Default | Description |
|---|---|---|
| `-wd\|--working-directory` | current dir | Repository root |
| `-a\|--dotnet-build-arguments` | — | Extra arguments forwarded to `dotnet build` |
| `-n\|--build-name` | — | Named build; status is persisted by `IRepositoryBuildStatusStore` |
| `-f\|--force` | false | Build all projects regardless of changes |

<Note>
When a `.sln`/`.slnx` file path is present in the build config, the command builds the full solution with `DotNetProjectBuilder.BuildSolution` instead of the changed-project mode.
</Note>

## `clean`

**Class:** `CleanCommand`

Runs `dotnet clean` on the current directory, then recursively deletes all `bin` and `obj` directories (skipping paths inside `node_modules`).

```
abp clean
```

No options. Useful before switching branches or when debugging stale build artifacts.

## `login`

**Class:** `LoginCommand`

Authenticates the local machine with `account.abp.io`. The access token is stored locally via `AuthService` and is used by commands that access Pro content (template downloads, module source code).

```
abp login <username>
abp login <username> -p <password>
abp login --device
```

| Flag | Description |
|---|---|
| `-p\|--password` | Password (prompted interactively if omitted) |
| `--device` | Device-code flow — opens a browser; required for accounts with 2FA |
| `-o\|--organization` | Organization slug (required for multi-org accounts) |

---

## `logout`

**Class:** `LogoutCommand`

Removes the stored access token. No options.

```
abp logout
```

---

## `list-modules`

**Class:** `ListModulesCommand`

Fetches the module catalog from abp.io via `ModuleInfoProvider` and prints a formatted table. Free modules are always shown; Pro modules are hidden unless `--include-pro-modules` is passed.

```
abp list-modules
abp list-modules --include-pro-modules
```

---

## `list-templates`

**Class:** `ListTemplatesCommand`

Calls the abp.io templates API and prints each template's `Name`, `DisplayName`, and `DocumentUrl`.

```
abp list-templates
```

## `get-source`

**Class:** `GetSourceCommand`

Downloads the complete source code of an ABP module (open-source or Pro) as a ZIP and extracts it locally. Uses `SourceCodeDownloadService.DownloadModuleAsync` which handles authentication for Pro modules.

```
abp get-source <module-name> [options]
```

| Flag | Default | Description |
|---|---|---|
| `-o\|--output-folder` | current dir | Where to place the extracted source |
| `-v\|--version` | latest | Specific ABP version to download |
| `--preview` | false | Use latest pre-release version |
| `--abp-path` | — | Override with a local ABP repository clone |
| `--volo-path` | — | Override with a local Volo repository clone |

---

## `update`

**Class:** `UpdateCommand`

Updates all ABP NuGet packages in a solution to the latest (or a specified) version. Internally uses `VoloNugetPackagesVersionUpdater`.

```
abp update [options]
```

| Flag | Description |
|---|---|
| `-v\|--version` | Target version; omit for latest stable |
| `--preview` | Use latest pre-release |
| `--npm` | Also update ABP NPM packages |
| `--stable` | Force stable channel even if on a preview install |
| `-s\|--solution` | Path to the `.sln` file |
| `-sp\|--startup-project` | Startup project for migration commands |
| `--check-all` | Check all packages, not just ABP ones |

## `install-libs`

**Class:** `InstallLibsCommand`

Installs client-side library dependencies defined in `abp.resourcemapping.js` (for MVC/Razor Pages projects) or runs `npm install` + `abp install-libs` for Angular projects.

## `cli`

**Class:** `CliCommand`

Updates or removes the ABP CLI global tool itself.

```
abp cli update
abp cli update --preview
abp cli update --version 9.0.0
abp cli remove
```

| Sub-command | Description |
|---|---|
| `update` | Updates the `Volo.Abp.Cli` global tool to the latest (or specified) version |
| `remove` | Uninstalls the `Volo.Abp.Cli` global tool |

| Flag | Description |
|---|---|
| `--preview` | Use the latest pre-release version |
| `--version` | Specific version to install |

---

## `mcp`

**Class:** `McpCommand`

Starts a local [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) server that exposes ABP tooling to AI assistants (e.g. Claude, Cursor). Requires an active ABP license. The command does **not** print a version banner and suppresses all non-JSON output to avoid corrupting the JSON-RPC stdout stream.

```
abp mcp
abp mcp get-config
```

| Sub-command / option | Description |
|---|---|
| _(no argument)_ | Start the local MCP server process |
| `get-config` | Print the MCP client configuration JSON to stdout |

The server connects to the ABP.IO MCP backend. If the server is unreachable the command exits with an error.

## `generate-jwks`

**Class:** `GenerateJwksCommand`

Generates an RSA key pair for use with OpenIddict `private_key_jwt` client authentication. Produces two files: a JWKS JSON file (public key only) and a PEM private key file.

```
abp generate-jwks [options]
```

| Flag | Default | Description |
|---|---|---|
| `-o\|--output` | current dir | Output directory |
| `-s\|--key-size` | `2048` | RSA key size: `2048` or `4096` |
| `--alg` | `RS256` | Algorithm: `RS256`, `RS384`, `RS512`, `PS256`, `PS384`, `PS512` |
| `--kid` | auto-generated | Key ID (`kid`) field |
| `-f\|--file` | `jwks` | File name prefix; generates `<prefix>.json` and `<prefix>-private.pem` |

Paste the generated JWKS public key into the ABP OpenIddict application's **JSON Web Key Set** field in the management UI. Store the PEM private key securely — never commit it to source control.

  </Card>

  </Card>

  </Card>
  <Card title="prompt" icon="terminal">

  </Card>
</CardGroup>

<Warning>
The following commands are decorated with `[HideFromCommandList]` and will not appear in `abp help` output, but they are fully functional and registered in `AbpCliOptions.Commands`:

- `recreate-initial-migration` (`RecreateInitialMigrationCommand`)
- `create-migration-and-run-migrator` (`CreateMigrationAndRunMigratorCommand`) — used internally after `abp new` to set up initial EF Core migrations and run the `DbMigrator`

`generate-razor-page` (`GenerateRazorPage`) is also registered in the command table and is not marked hidden, but is considered an advanced/internal command not covered in the main help text.
</Warning>
